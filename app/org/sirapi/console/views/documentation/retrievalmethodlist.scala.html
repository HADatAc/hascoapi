Elements Retrieval RESTful API


Retrieval of SIR Elements

Note:Every single SIR element is the instantiation of a concept, which is registered at SIR under an unique URI. Instances of concepts like instruments, instrument types, and detectors are example of SIR elements.

- Returns a full description of an element (instrument, detector, codebook or response option) as described in SIR. URI needs to be URL encoded. Use the Query URI Example to query different kinds of URI Elements
- Returns elements that make use of the element with the provided uri (e.g., instruments that make use of a given detector). URI needs to be URL encoded. Use the Query URI Example to query different kinds of URI Elements
- Returns elements that are derived from the element with the provided uri (e.g., detectors derived from a given detector). URI needs to be URL encoded. Use the Query URI Example to query different kinds of URI Elements
- Returns elements in SIR of the given type that simultaneously have the keyword as part of their labels and that are in the requested language, including their properties.
- Returns the number of elements in SIR of the given type that simultaneously have the keyword as part of their labels and that are in the requested language, including their properties.
- Returns elements in SIR of the given type that are managed by the user with the given manager email.
- Returns the number of elements in SIR of the given type that are managed by the user with the given manager email.
- Returns elements in SIR that make use of the element of the given element uri.
GET    /sirapi/api/instrument/all - Returns all instruments in SIR, including their properties.
- Returns instruments in SIR that have the keyword as part of their labels, including their properties.
- Returns instruments in SIR that are in the requested language, including their properties.
GET    /sirapi/api/instrumenttype/all - Returns all instrument types currently loaded into SIR.
GET    /sirapi/api/detectorSlot/all - Returns all detectorSlots in SIR, including their properties.
- Returns detectorSlots that belong to an instrument with the given URI. URI needs to be URL encoded.
GET    /sirapi/api/detector/all - Returns all detectors in SIR, including their properties.
- Returns detectors in SIR that have the keyword as part of their labels, including their properties.
- Returns detectors in SIR that are in the requested language, including their properties.
- Returns detectors that belong to an instrument with the given URI. URI needs to be URL encoded.
GET    /sirapi/api/detectortype/all - Returns all detector types currently loaded into SIR.
GET    /sirapi/api/codebook/all - Returns all codebooks currently loaded into SIR.
- Returns codebooks in SIR that have the keyword as part of their labels, including their properties.
- Returns codebooks in SIR that are in the requested language, including their properties.
GET    /sirapi/api/responseoption/all - Returns all responseoptions currently loaded into SIR.
- Returns responseoptions in SIR that have the keyword as part of their labels, including their properties.
- Returns responseoptions in SIR that are in the requested language, including their properties.
- Returns responseoptions that belong to an codebook with the given URI. URI needs to be URL encoded.

Interoperability of SIR Elements


Semantic Support for Instrument Interoperability

- Returns RDF encoding in XML for requested instrument.
- Returns FHIR encoding in JSON for requested instrument.
- Returns RedCAP encoding for requested instrument.

Semantic Support for Instrument Response Interoperability

- Returns Semantic Data Dictionary of instrument response for requested instrument.

Instrument Rendering (for Instruments that are not Physical Devices)

Note: Use the Instrument Rendering Examples to see how to render the Test Instrument using different kinds of formats.
- Returns Text encoding in Plain for requested instrument.
- Returns Text encoding in HTML for requested instrument.
- Returns Text encoding in PDF for requested instrument.
