Elements Retrieval RESTful API


Retrieval of SIR Elements

Note:Every single SIR element is the instantiation of a concept, which is registered at SIR under an unique URI. Instances of concepts like instruments, instrument types, and detectors are example of SIR elements.

- Return a full description of a concept as described in SIR. URI needs to be URL encoded. Use the Query URI Example to query different kinds of URI Elements
GET    /sirapi/api/instrument/all - Return all instruments in SIR, including their properties.
- Return instruments in SIR that have the keyword as part of their labels, including their properties.
- Return instruments in SIR that are in the requested language, including their properties.
- Return instruments in SIR that simultaneously have the keyword as part of their labels and that are in the requested language, including their properties.
- Return instruments in SIR that have the requested maintainer email.
GET    /sirapi/api/instrumenttype/all - Return all instrument types currently loaded into SIR.
GET    /sirapi/api/detector/all - Return all detectors in SIR, including their properties.
- Return detectors in SIR that have the keyword as part of their labels, including their properties.
- Return detectors in SIR that are in the requested language, including their properties.
- Return detectors in SIR that simultaneously have the keyword as part of their labels and that are in the requested language, including their properties.
- Return detectors in SIR that have the requested maintainer email.
- Return detectors that belong to an instrument with the given URI. URI needs to be URL encoded.
GET    /sirapi/api/detectortype/all - Return all detector types currently loaded into SIR.
GET    /sirapi/api/experience/all - Return all experiences currently loaded into SIR.
- Return experiences in SIR that have the keyword as part of their labels, including their properties.
- Return experiences in SIR that are in the requested language, including their properties.
- Return experiences in SIR that simultaneously have the keyword as part of their labels and that are in the requested language, including their properties.
- Return experiences in SIR that have the requested maintainer email.
GET    /sirapi/api/responseoption/all - Return all responseoptions currently loaded into SIR.
- Return responseoptions in SIR that have the keyword as part of their labels, including their properties.
- Return responseoptions in SIR that are in the requested language, including their properties.
- Return responseoptions in SIR that simultaneously have the keyword as part of their labels and that are in the requested language, including their properties.
- Return responseoptions in SIR that have the requested maintainer email.
- Return responseoptions that belong to an experience with the given URI. URI needs to be URL encoded.

Interoperability of SIR Elements


Semantic Support for Instrument Interoperability

- Return RDF encoding in Turtle for requested instrument.
- Return RDF encoding in OWL/XML for requested instrument.
- Return FHIR encoding for requested instrument.
- Return RedCAP encoding for requested instrument.

Semantic Support for Instrument Response Interoperability

- Return Semantic Data Dictionary of instrument response for requested instrument.

Instrument Rendering (for Instruments that are not Physical Devices)

Note: Use the Instrument Rendering Examples to see how to render the Test Instrument using different kinds of formats.
- Return Text encoding in Plain for requested instrument.
- Return Text encoding in HTML for requested instrument.
- Return Text encoding in PDF for requested instrument.
