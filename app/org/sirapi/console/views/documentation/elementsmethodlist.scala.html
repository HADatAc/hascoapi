Element Management RESTful API


Maintenance of SIR Elements

Note: Instrument types and detector types are loaded into SIR through the VSTOI ontology. The VSTOI ontology needs to be modified and reloaded to create instrument type and detector type or to delete an existing instrument type or detector type.


Instruments

- Create a new instrument according to the json specification provided. Json needs to be URL encoded. Use the Create Instrument Example to see how to use a CURL with POST method to create instruments.
- Delete an existing instrument using the URI provided. URI needs to be URL encoded. Use the Delete Instrument Example to see how to use a CURL with POST method to delete instruments.
- For an existing instrument with the URI provided, create the given number of attachments.
- For an existing instrument with the URI provided, delete all its attachment at once.

Detectors

- Create a new detector according to the json specification provided. Json needs to be URL encoded. Use the Create Detector Example to see how to use a CURL with POST method to create detectors.
- Delete an existing detector using the URI provided. URI needs to be URL encoded. Use the Delete Instrument Example to see how to use a CURL with POST method to delete detectors.
- Attach a detector to an instrument's attachment. This method requires the detector and instrument's attachment to exist.
- Detach a detector from a given instrument's attachment.

Experiences

- Create a new experience according to the json specification provided. Json needs to be URL encoded.
- Delete an existing experience using the URI provided. URI needs to be URL encoded.

Response Options

- Create a new response option according to the json specification provided. Json needs to be URL encoded.
- Delete an existing response option using the URI provided. URI needs to be URL encoded.

Testing of SIR Elements


Test Instrument

GET    /sirapi/api/instrument/test/create - Create a test instrument.
GET    /sirapi/api/instrument/test/delete - Delete a test instrument.
GET    /sirapi/api/attachment/test/create - Create two attachments for the test instrument. This methods requires the test instrument to exist before it is called.
GET    /sirapi/api/attachment/test/delete - Delete attachments of test instrument, if any. This method requires the test instrument to exist and to have attachments.

Test Detector

Note: Create a test instrument along with its attachments before creating test detectors

GET    /sirapi/api/detector/test/create - Create test detector 1 and 2 for a test instrument.
GET    /sirapi/api/detector/test/delete - Delete test detector 1 and 2.
GET /sirapi/api/detector/test/attach - Attach test detectors 1 and 2 to test instrument.
GET /sirapi/api/detector/test/detach - Detach test detectors 1 and 2 from test instrument.

Test Experiment

GET    /sirapi/api/experience/test/create - Create test experiment.
GET    /sirapi/api/experience/test/delete - Delete test experiment.

Test Response Option

Note: Create a test experience before creating test response options

GET    /sirapi/api/responseoption/test/create - Create test response options 1 and 2 for a test instrument.
GET    /sirapi/api/responseoption/test/delete - Delete test response options 1 and 2.
