Element Management RESTful API

Note: Instrument types and detector types are loaded into SIR through the VSTOI ontology. The VSTOI ontology needs to be modified and reloaded to create instrument type and detector type or to delete an existing instrument type or detector type.


Instruments

- Create a new instrument according to the json specification provided. Json needs to be URL encoded. Use the Create Instrument Example to see how to use a CURL with POST method to create instruments.
- Delete an existing instrument using the URI provided. URI needs to be URL encoded. Use the Delete Instrument Example to see how to use a CURL with POST method to delete instruments.
- For an existing instrument with the URI provided, create the given number of detectorSlots.
- For an existing instrument with the URI provided, delete all its detectorSlot at once.

Detector Stems

- Create a new detector stem according to the json specification provided. Json needs to be URL encoded. Use the Create Detector StemExample to see how to use a CURL with POST method to create detector stems.
- Delete an existing detector stem using the URI provided. URI needs to be URL encoded. Use the Delete Detector Stem Example to see how to use a CURL with POST method to delete detector stems.

Detectors

- Create a new detector according to the json specification provided. Json needs to be URL encoded. Use the Create Detector Example to see how to use a CURL with POST method to create detectors.
- Delete an existing detector using the URI provided. URI needs to be URL encoded. Use the Delete Instrument Example to see how to use a CURL with POST method to delete detectors.
- Attach a detector to an instrument's detectorSlot. This method requires the detector and instrument's detectorSlot to exist.
- Detach a detector from a given instrument's detectorSlot.

Codebooks

- Create a new codebook according to the json specification provided. Json needs to be URL encoded.
- Delete an existing codebook using the URI provided. URI needs to be URL encoded.
- For an existing codebook with the URI provided, create the given number of responseOptionSlots.
- For an existing codebook with the URI provided, delete all its responseOptionSlots at once.

Response Options

- Create a new response option according to the json specification provided. Json needs to be URL encoded.
- Delete an existing response option using the URI provided. URI needs to be URL encoded.
