Element Management RESTful API

Note: Instrument types and detector types are loaded into SIR through the VSTOI ontology. The VSTOI ontology needs to be modified and reloaded to create instrument type and detector type or to delete an existing instrument type or detector type.


Generic

Methods for following element types: instrumenttype, instrument, subcontainer, detectorstemtype, detectorstem, detector, codebook, responseoption, annotationstem, annotation, semanticvariable, entity, attribute or unit

- Create a new object of given element type according to the json specification provided. Json needs to be URL encoded. Use the Create Instrument Example to see how to use a CURL with POST method to create instruments.
- Delete an existing object of given element using the URI provided. URI needs to be URL encoded. Use the Delete Instrument Example to see how to use a CURL with POST method to delete instruments.

ContainerSlot-Specific Methods

- For an existing container (instrument or subcontainer) with the URI provided, create the given number of containerSlots.
- For an existing container (instrument or subcontainer) with the URI provided, delete all its containerSlot at once.

Detectors-Specific Methods

- Attach a detector to an instrument's containerSlot. This method requires the detector and instrument's containerSlot to exist.
- Detach a detector from a given instrument's containerSlot.

CodebookSlot-Specific Methods

- For an existing codebook with the URI provided, create the given number of codebookSlots.
- For an existing codebook with the URI provided, delete all its codebookSlots at once.

ResponseOption-Specific Methods

- Attach a responseOption to a codebook's codebookSlot. This method required the response option and codebook's codebookSlot to exist.
- Detach a responseOption from a given codebook's codebookSlot.
